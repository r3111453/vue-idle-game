# 游戏设计

- 通过副本获得装备与金币
- 装备分为四个等级：破旧、普通、神器、史诗
- 主角默认每秒回血 2%
- 默认暴击伤害 150%
- 完成副本时可能掉落独特级别(5 个词条)的装备（通关后 2.5%掉落率，无尽挑战不会掉落）
- 通关后开启无尽挑战

# 更新内容

## 2026-4-20 更新 (1.3.7)

#### 优化

- 商店免费刷新次数现在会保存在浏览器中（`localStorage`），刷新页面（F5）不再重置剩余次数
- 页面首次加载时，商店商品初始化不再消耗免费刷新次数
- 移除 Q&A 组件中向外部服务器（`http://couy.xyz:3001`）的网络请求，彻底消除 Mixed Content 安全警告

#### bug 修复

- 已修复：自动购买独特装备时，现在只过滤不可洗的基础词条（固定属性），可洗的额外词条不再影响购买判断
- 已修复：商店因 `localStorage` 读取到异常值（如0）导致商品无法显示的问题
- 已修复：页面刷新后剩余刷新次数错误减少的问题

## 2026-4-19 更新 (1.3.6)

#### 优化

- 反馈表单提交冷却时间调整为10分钟，冷却状态保存在浏览器中（刷新页面不会重置）
- 提交按钮在冷却期间自动禁用，并显示剩余秒数提示
- 增加蜜罐字段（`_gotcha`），用于拦截自动化脚本的垃圾提交
- 配合 Formspree 的 IP 速率限制和域名白名单，有效防止恶意刷量

#### bug 修复

- 已修复：反馈表单可能被短时间内重复提交的问题
- 已修复：商店自动购买独特装备时，“基础属性最低百分比”阈值判断无效的问题（例如设置100%仍会买下暴伤+78%的装备）

## 2026-4-18 更新 (1.3.5)

#### 优化

- 背包一键整理现在支持排序模式切换（可选择按类型、品质、等级排序，或选择不排序）
- 血量恢复后自动重新挑战地下城功能
- 自动刷新和购买商店装备
- 背包容量提升到100，除此之外未对原版参数进行任何修改
- 地下城及其怪物强度、生命、攻击力显示，计算能否胜利和恢复时长
- 装备属性强度显示
- 属性面板计算公式显示
- 商店装备属性直接显示
- 强化界面估算自动强化到目标等级的消耗显示
- 存档保存背包自动出售的设置
- 微调部分界面

# 本地运行

### 安装依赖

```sh
  npm install
```

### 启动项目

```sh
  npm run serve
```

### 打包项目

```sh
  npm run build
```
